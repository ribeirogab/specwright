memex benchmark · reference

tlc-spec-driven — Tech Lead's Club spec-driven skill

memex's closest sibling: a single markdown Agent Skill that drives an agent through a complexity-adaptive pipeline — Specify → Design → Tasks → Execute — auto-sizing its own ceremony, threading requirement-traceability IDs, and baking test-first discipline into every task. Like memex, all rigor is prose — zero machine enforcement.

Medium

1 SKILL.md (216 lines) + 16 references/*.md. Pure prose & templates.

Core idea

"The complexity is in the system, not in your workflow." Scope decides depth.

Memory

On-disk .specs/ vault. STATE.md = durable brain (AD/B/L entries).

Enforcement

prose only No validator, no CLI gate. Model-compliance dependent.

Safety valve: even when Tasks is skipped, Execute lists atomic steps first; if that reveals >5 steps, it STOPs and creates a real tasks.md. Auto-skip self-corrects.

What makes it rigorous (all prose)

  • Traceability IDs — AUTH-01 threaded spec→design→task→commit→validation, with a coverage line X total, Y mapped, Z unmapped ⚠️.
  • Three pre-approval checks in Tasks (granularity, diagram-definition cross-check, test co-location) — two emitted as tables before the user sees anything.
  • Test co-location — tests live in the same task as the code; "tested in another task" is banned.
  • RED→GREEN with hard constraints: never modify/weaken/skip/delete the RED tests.
  • Test-integrity checks — test count must not silently drop; weakened assertions = regression. targets AI failure modes
  • Knowledge Verification Chain — Codebase → Project docs → Context7 → Web → flag uncertain. Absolute anti-fabrication rule.

Strengths & limitations

Strengths

  • Genuinely adaptive — one tool for a one-liner and a big feature.
  • Concrete AI-aware rigor (traceability, test-integrity, anti-fabrication).
  • Externalized human-readable memory with token budgets.
  • First-class sub-agent delegation, parallelism gated by test-safety.

Limitations

  • No machine enforcement — every "MANDATORY" is prose.
  • Auto-sizing relies on the agent's own scope judgment.
  • No branch / PR / mode concept — stops at atomic commits.
  • Effectiveness is explicitly model-dependent.

vs memex

TLC has, memex lacks: graded tiers + safety valve · traceability IDs · test machinery (coverage matrix, parallelism, co-location, integrity checks) · interactive UAT · anti-fabrication chain.

memex has, TLC lacks: branch + PR discipline · autonomous/reviewed mode · spec self-review vs a constitution · constitution+rules law layer · wikilinked learnings vault + reflection + GC.

Net: TLC is deeper on per-task engineering rigor; memex is deeper on lifecycle governance.
