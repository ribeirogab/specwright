# Conventions — Map of Content

Deliberate code style choices that all code in agent-skills must follow. These are not safety rules (those live in the constitution) and not things learned from incidents (those live in learnings). These are team decisions about how code should look and be structured.

## Skill authoring

- [[../conventions/skill-directory-layout|Skill directory layout]] — canonical `SKILL.md` + `scripts/` + `references/` + `assets/`; one level deep; no `README.md` inside; folder name = `name` field.
- [[../conventions/skill-md-style|SKILL.md authoring style]] — description field (what + when + negatives, third person), gerund naming, body <500 lines, consistent terminology, forward slashes, MCP tools fully qualified.
