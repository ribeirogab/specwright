# Resume — Learnings

Curated facts downstream issues inherit via their specs.

- Milestone state discovery from files alone is confirmed working in both entry modes: `/sw:run` with no arguments (no-arg scan found `2026-07-02-grow-taskr` as the single milestone with unshipped issues) and natural language — and the NL trigger matches by *intent*, not exact text ("continue the **taskr** milestone" hit the skill's 'continue the milestone' trigger). Downstream issues can rely on either entry without passing paths.
- An orchestrator session framed in approval mode (any non-read action needs user approval) yields exactly once before dispatch, because `/sw:run` batches *all* of round 1's writes — worktree add + owner dispatch + Dispatch Log append — behind a single approval ask. Corollary for dispatch-parallelism (T4) and beyond: an unattended (auto-approve) run has **no natural pause** between announcing the ready set and writing/dispatching; harnesses that need a pre-dispatch stop must supply an approval gate themselves.
- Holding at that approval ask leaves the board **fully** byte-identical — even the Dispatch Log stays empty, because the log append is part of the batched dispatch action, not of discovery. "Legitimately logs before holding" turned out to be the empty set.
- The T1 driver-identity leak reproduced: driven sessions see the driver's agent name via SendMessage sender attribution (here `owner-resume`, which carries the issue slug). The leak mechanism observed so far is SendMessage-only (spawn prompts carry no sender name, and both sessions' first turns referenced no caller identity), but spawn-time metadata is not inspectable — spawn drivers under a neutral name instead of relying on the timing argument.
- The driver's global user instructions propagate into driven sub-agent sessions (both sessions answered in pt-BR despite an all-English framing prompt). Content and behavior were unaffected; transcript graders must not read the language as a divergence.
- Verbatim turn capture without relayed paraphrase: the sub-agent's JSONL output file can be queried surgically with `jq` (`select(.type=="assistant") | .message.content[] | select(.type=="text") | .text`) — exact turn text for evidence, no context overflow, satisfying the T2 verbatim-evidence lesson.
