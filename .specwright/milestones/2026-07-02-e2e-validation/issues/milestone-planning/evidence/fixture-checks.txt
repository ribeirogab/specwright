11:Grow taskr beyond `add`/`list`/`done`: tasks gain a priority (high/medium/low), the list becomes filterable by priority and by status, tasks can be exported to JSON and CSV with ISO-8601 dates, a minimal read-only web page shows the tasks and their status, and `taskr list` shows the newest task first by default.
15:taskr covers only the bare add/list/done loop. The maintainer needs to triage by priority, slice the list by state, feed the data to other tools (JSON/CSV with standard dates), and glance at the tasks in a browser — all without giving up taskr's zero-dependency simplicity.
19:- A task can be created with a priority, listed filtered by priority and/or status, seen on a local web page with its status, and exported to both JSON and CSV with ISO-8601 dates — end to end, using only Node and the taskr CLI.
20:- `npm test` is green on every issue's PR with the pre-existing `test/taskr.test.js` byte-for-byte untouched.
21:- `package.json` still declares zero runtime dependencies.
25:- Touching `test/taskr.test.js` in any way (the hard constraint; new behavior gets new test files).
26:- Migrating or rewriting the store file format (`createdAt` stays epoch seconds; tasks without `priority` stay valid and are read as `medium`).
