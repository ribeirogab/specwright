# Milestone Planning — Learnings

Curated facts downstream issues inherit via their specs.

- `validate-spec.sh` on a planning-stage issue folder (only `issue.md`, no `spec.md`/`tasks.md`) exits 1 with exactly one `FAIL (check 2): spec.md not found` line — that is the expected baseline, not a defect; only additional FAIL lines are real. Any gate or audit running the validator against undispatched tickets must apply this interpretation.
- The grow-taskr fixture ships with a live mechanical defect downstream issues will hit: `task-priority`'s AC-2 contains "(short alias works)" and trips validator check 4 (banned word "works"). When that issue's owner runs `/sw:plan`, the mechanical gate fails on a file the owner must not unilaterally edit (AC changes are scope changes) — dispatch-parallelism (T5) and any issue-owner run over task-priority should expect a report/blocked path there, not thrashing (findings.md Divergence 3.1).
- goal.md in the fixture carries technical content (file paths `test/taskr.test.js` and `package.json` in Success Criteria/Non-Goals, `createdAt` epoch-seconds storage format) — a planning-contract FAIL charged to the brainstorm skill's milestone-writing guidance, not to the fixture (findings.md Divergences 1.1/1.2). board.md and all five tickets are otherwise contract-clean.
- Relayed transcripts cannot prove handoff wording: the scope-detection evidence relays that a handoff was printed but not its text, leaving "handoff names the milestone path" unverifiable. Session-driving issues should capture the driven session's final handoff verbatim into evidence (ask the session to print it as its last message, or have the driver quote it).
- Convention tension for the fixes milestone: `vault-files.md` line 58 says sibling issues carry "no cross-references between issues", but evidence-consuming audit issues (this one) must cite sibling evidence paths (`../scope-detection/evidence/...`) to stay verifiable; line 95 only bans link syntax. Propose a carve-out for evidence-consuming issues instead of dropping the citations.
